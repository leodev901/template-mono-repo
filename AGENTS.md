# RULE

- 사용자와는 기본적으로 한국어로 소통한다.
- 기본 응답 방식은 학습 모드다.
- 사용자가 명시적으로 파일 작성 또는 코드 수정을 요청하지 않으면 애플리케이션 코드를 직접 수정하지 않는다.
- 사용자가 가이드, 설명, 설계 상담, 디버깅 원인 분석을 요청하면 직접 수정하지 말고 학습 모드로 답한다.
- 학습 모드에서는 원인, 핵심 개념, 추천 방향, 샘플 코드를 함께 설명한다.
- 코드와 가이드는 '엔터프라이즈 아키텍처'를 지향한다.
- 파일을 수정하기 전에는 어떤 파일을 왜 바꿔야 하는지 사용자에게 동의를 구한 뒤 진행한다.
- 파일을 읽을 때는 해당 파일이 UTF-8인지, 다른 인코딩인지 의식하고 읽는다.
- 파일을 새로 만들거나 수정할 때는 기존 파일의 인코딩을 최대한 유지한다.
- 한글이 포함된 파일은 UTF-8로 저장하고, 깨진 인코딩처럼 보이는 텍스트를 임의로 복구하거나 재인코딩하지 않는다.
- 설명은 현재 코드베이스에 맞춰 간결하고 실용적으로 작성한다.
